# OpenCode에 기여하기

OpenCode에 쉽게 기여할 수 있도록 하고자 합니다. 가장 자주 병합되는 변경 유형은 다음과 같습니다:

- 버그 수정
- 추가 LSP / 포매터
- LLM 성능 개선
- 새 공급자 지원
- 환경별 이슈 수정
- 누락된 표준 동작
- 문서 개선

단, UI 또는 핵심 제품 기능은 구현 전에 코어 팀과의 디자인 리뷰를 거쳐야 합니다.

## Pull Request 기대사항

### Issue 우선 정책

**모든 PR은 기존 issue를 참조해야 합니다.** PR을 열기 전에 버그나 기능을 설명하는 issue를 먼저 열어주세요. 메인테이너의 분류를 돕고 중복 작업을 방지합니다. 연결된 issue가 없는 PR은 리뷰 없이 닫힐 수 있습니다.

### AI가 생성한 장황한 텍스트 금지

길고 AI가 생성한 PR 설명과 issue는 허용되지 않으며 무시될 수 있습니다. 메인테이너의 시간을 존중하세요:

- 짧고 집중된 설명을 작성하세요
- 변경 사항과 이유를 자신의 말로 설명하세요
- 짧게 설명할 수 없다면 PR이 너무 클 수 있습니다

### PR 제목

PR 제목은 conventional commit 표준을 따라야 합니다: `feat:` 새 기능, `fix:` 버그 수정, `docs:` 문서, `chore:` 유지보수, `refactor:` 리팩토링, `test:` 테스트.
