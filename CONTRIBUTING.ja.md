# OpenCode への貢献

OpenCode への貢献をしやすくしたいと考えています。最もよくマージされる変更の種類は以下のとおりです：

- バグ修正
- 追加の LSP / フォーマッタ
- LLM パフォーマンスの改善
- 新しいプロバイダのサポート
- 環境固有の癖への修正
- 欠けている標準動作
- ドキュメントの改善

ただし、UI またはコアの製品機能は、実装前にコアチームによるデザインレビューを通過する必要があります。

## プルリクエストへの期待

### Issue 優先ポリシー

**すべての PR は既存の issue を参照しなければなりません。** PR を開く前に、バグや機能を説明する issue を開いてください。これによりメンテナのトリアージが容易になり、重複作業を防ぎます。リンクされた issue がない PR はレビューなしでクローズされることがあります。

### AI が生成した長文は禁止

長い AI 生成の PR 説明や issue は受け入れられず、無視されることがあります。メンテナの時間を尊重してください：

- 短く焦点を絞った説明を書いてください
- 何が変わったのかと理由を自分の言葉で説明してください
- 簡潔に説明できなければ、PR が大きすぎるかもしれません

### PR タイトル

PR タイトルは conventional commit 標準に従う必要があります：`feat:` 新機能、`fix:` バグ修正、`docs:` ドキュメント、`chore:` メンテナンス、`refactor:` リファクタリング、`test:` テスト。
