# 為 OpenCode 貢獻

我們希望讓您更容易為 OpenCode 做出貢獻。以下是最常合併的變更類型：

- Bug 修復
- 額外的 LSP / 格式化器
- LLM 效能改進
- 支援新的提供商
- 針對特定環境怪癖的修復
- 缺失的標準行為
- 文件改進

但任何 UI 或核心產品特性都需要在實作之前與核心團隊進行設計審查。

## Pull Request 期望

### Issue 優先政策

**所有 PR 必須引用現有的 issue。** 開啟 PR 前，請先開 issue 描述 bug 或特性。這有助於維護者分流並防止重複工作。未連結 issue 的 PR 可能會不經審查而被關閉。

### 禁止 AI 生成的長篇大論

冗長的 AI 生成的 PR 描述和 issue 不可接受，可能被忽略。請尊重維護者的時間：

- 撰寫簡短、聚焦的描述
- 用自己的話解釋發生了什麼變化以及為什麼
- 若無法簡短解釋，您的 PR 可能太大

### PR 標題

PR 標題應遵循常規提交標準：`feat:` 新特性，`fix:` bug 修復，`docs:` 文件，`chore:` 維護，`refactor:` 重構，`test:` 測試。
