# Doprinos OpenCode-u

Želimo vam olakšati doprinos OpenCode-u. Evo najčešće mergovanih tipova promjena:

- Ispravke bugova
- Dodatni LSP-ovi / formatteri
- Poboljšanja LLM performansi
- Podrška za nove pružaoce
- Ispravke za posebnosti okruženja
- Nedostajuće standardno ponašanje
- Poboljšanja dokumentacije

Međutim, svaka UI ili osnovna proizvodna funkcija mora proći dizajn pregled s osnovnim timom prije implementacije.

## Očekivanja za Pull Requestove

### Politika Issue prvo

**Svi PR-ovi moraju referencirati postojeći issue.** Prije otvaranja PR-a, otvorite issue koji opisuje bug ili funkciju. To pomaže održavačima u triage-u i sprječava dupli rad. PR-ovi bez povezanog issue-a mogu biti zatvoreni bez pregleda.

### Bez zidova teksta generisanih AI-em

Dugi AI-generisani opisi PR-ova i issue-a nisu prihvatljivi i mogu biti ignorisani. Poštujte vrijeme održavača:

- Pišite kratke, fokusirane opise
- Objasnite šta se promijenilo i zašto svojim riječima
- Ako ne možete objasniti ukratko, vaš PR je možda prevelik

### Naslovi PR-ova

Naslovi PR-ova trebaju slijediti standard conventional commit: `feat:` nova funkcija, `fix:` ispravka buga, `docs:` dokumentacija, `chore:` održavanje, `refactor:` refaktorisanje, `test:` testovi.
