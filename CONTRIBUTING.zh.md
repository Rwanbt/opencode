# 为 OpenCode 贡献

我们希望让您更容易为 OpenCode 做出贡献。以下是最常合并的更改类型：

- Bug 修复
- 额外的 LSP / 格式化器
- LLM 性能改进
- 支持新的提供商
- 针对特定环境怪癖的修复
- 缺失的标准行为
- 文档改进

但任何 UI 或核心产品特性都需要在实现之前与核心团队进行设计评审。

## Pull Request 期望

### Issue 优先政策

**所有 PR 必须引用现有的 issue。** 开启 PR 前，请先开 issue 描述 bug 或特性。这有助于维护者分流并防止重复工作。未链接 issue 的 PR 可能会不经审查而被关闭。

### 禁止 AI 生成的长篇大论

冗长的 AI 生成的 PR 描述和 issue 不可接受，可能被忽略。请尊重维护者的时间：

- 编写简短、聚焦的描述
- 用自己的话解释发生了什么变化以及为什么
- 若无法简短解释，您的 PR 可能太大

### PR 标题

PR 标题应遵循常规提交标准：`feat:` 新特性，`fix:` bug 修复，`docs:` 文档，`chore:` 维护，`refactor:` 重构，`test:` 测试。
